# Evaluation Protocol

## Split strategy

Use a subject-wise split. Every raw sample and every downstream window from a given `subject_id` belongs to exactly one split.

This avoids identity leakage: adjacent time-series samples from the same person are highly correlated, and a naive random row split would put near-duplicate movement patterns from the same subject into both train and test sets.

Default split targets are approximately 70% train, 15% validation, and 15% test by subject. The script searches deterministic seeds until each split has coverage of all selected activity labels when possible.

## Recommended reporting

- During model selection, fit preprocessing scalers and models on train windows only.
- Use validation only for model choice and hyperparameter tuning.
- After choosing a baseline configuration, optionally refit on train plus validation before the single held-out test evaluation.
- Report final accuracy, macro F1, confusion matrix, and per-class metrics on the held-out test subjects.
- For more robust estimates, use grouped cross-validation by `subject_id` as a secondary analysis.
